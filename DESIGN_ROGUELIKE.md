# Roguelike 升级选择 + BOSS 系统 — 完整方案设计

> 基于现有飞机大战项目（v5），引入 Roguelike 升级选择、精英敌机、BOSS 战等机制，增强可玩性与留存。
> 参考游戏：Vampire Survivors、王者荣耀守卫陈塘关、天空之翼、Brotato、HoloCure、Soulstone Survivors

---

## 一、核心架构变更

### 1.1 核心循环

**旧流程**：
```
击杀敌机 → 自动获得经验 → 升级 → 自动获得固定属性加成（HP/伤害/射速/Buff时长）
```

**新流程**：
```
击杀敌机 → 自动获得经验 → 升级 → 暂停游戏 → 弹出3选1升级界面 → 玩家选择 → 恢复游戏
```

### 1.2 关键设计决策

| 决策点 | 方案 | 理由 |
|--------|------|------|
| 升级选择是否暂停游戏 | **暂停**（时间冻结） | 竖屏射击节奏快，不暂停容易被打死；VS 也是暂停选择 |
| 现有固定属性加成是否保留 | **不保留**，全部转入升级池 | 升级选择的核心乐趣是"取舍"，固定加成与之矛盾 |
| 现有道具掉落是否保留 | **保留**，作为即时效果 | 道具（回血/临时火力/临时护盾/临时散弹）是战术层，升级是战略层 |
| Buff 系统是否保留 | **保留**，来源从"道具+升级"双渠道 | 道具给临时 Buff，升级给永久技能/武器 |
| 武器与 Buff 的关系 | 武器=永久自动攻击，Buff=临时增益 | 参考 VS：武器是"你有什么"，Buff 是"你暂时更强" |
| 现有难度系统 | **重构**，新增升级/BOSS 维度 | 旧乘数与新成长曲线冲突，需重新校准 |

### 1.3 新增游戏阶段

当前 `GamePhase` 有 6 个阶段，新增 1 个：

```typescript
export const PHASE_LEVEL_UP = 7 as const;  // 升级选择阶段（游戏暂停，等待玩家选择）
```

- 进入条件：`addExp()` 返回值 > 0（升级了）
- 退出条件：玩家点击选择了一张卡片
- 连续升级时：选完一个立即弹出下一个，不合并
- 在此阶段：背景和敌机静止不动但仍渲染（视觉冻结），UI 层显示选择界面

## 二、升级分类体系

升级分为三大类：**武器**、**被动**、**特殊**。

### 2.1 武器（Weapon）—— 永久自动攻击

每个武器有独立的冷却计时器，到时间自动发射。玩家最多拥有 **6 个武器槽位**（1 个基础武器 + 5 个额外武器）。

#### 基础武器（开局自带，可升级至 Lv5）

| 等级 | 效果 | 说明 |
|------|------|------|
| Lv1 | 三路子弹（当前默认） | 左/中/右 3 发 |
| Lv2 | 伤害 +30% |  |
| Lv3 | 四路子弹 | +1 发 |
| Lv4 | 伤害 +30% + 射速 +20% |  |
| Lv5 | 五路子弹 + 穿透 | 5 发齐射 + 子弹穿透 1 个敌机 |

#### 额外武器（升级时获得，自动发射，每个可升级至 Lv5）

| 武器 | 发射方式 | Lv1 效果 | Lv5 效果 | 视觉特征 | 稀有度 |
|------|---------|---------|---------|---------|--------|
| **追踪导弹** | 每 90 帧发射 | 1 枚导弹，追踪最近敌机，伤害 3 | 3 枚导弹，伤害 8，小范围爆炸 | 橙色拖尾曲线弹道 | 稀有 |
| **护盾环绕** | 持续旋转 | 1 个环绕球体，接触伤害 2 | 3 个球体，伤害 6，击退效果 | 蓝色发光球体围绕战机 | 普通 |
| **闪电链** | 每 120 帧触发 | 劈最近敌机，链 1 次，伤害 4 | 链 3 次，伤害 10，附带减速 | 白色锯齿线段+分支 | 稀有 |
| **爆裂弹** | 每 150 帧发射 | 1 枚慢速大弹，小范围爆炸，伤害 5 | 大范围爆炸，伤害 12，灼烧 DOT | 红色大弹+爆炸圆环 | 普通 |
| **激光炮** | 每 180 帧触发 | 短持续激光束，伤害 8 | 长激光束，伤害 20，穿透全部 | 白蓝色线段+发光 | 史诗 |
| **僚机** | 持续射击 | 1 架僚机在侧方，每 6 帧射 1 发，伤害 1 | 2 架僚机，伤害 3，共享射速加成 | 小型战机精灵 | 稀有 |
| **火焰光环** | 持续判定 | 近身小范围灼烧，伤害 1/帧 | 大范围 + 减速 30%，伤害 3/帧 | 红橙色半透明圆环 | 普通 |
| **冰冻弹** | 每 100 帧发射 | 1 发，减速 40% 持续 60 帧 | 2 发，冻结 30 帧，伤害 6 | 浅蓝色菱形弹+冰晶 | 稀有 |

### 2.2 被动（Passive）—— 永久属性提升

纯数值提升，可叠加，**不限数量**。

| 被动 | 效果 | 稀有度 | 说明 |
|------|------|--------|------|
| 生命强化 | Max HP +1（当前 HP 也 +1） | 普通 | 直接提升生存 |
| 伤害增幅 | 所有武器伤害 +15%（乘法叠加） | 普通 | 核心输出被动 |
| 射速提升 | 基础武器射速 +10%（减少射击间隔） | 普通 | 提升DPS |
| 移速提升 | 移动速度 +8% | 普通 | 更灵活走位 |
| 吸引范围 | 道具拾取范围 +30% | 普通 | 磁铁效果 |
| 暴击率 | +5% 暴击率（暴击 = 2× 伤害） | 稀有 | 所有武器共享 |
| 暴击伤害 | 暴击倍率 +0.5（基础 2× → 2.5×） | 稀有 | 前置：暴击率 |
| 经验加成 | 经验获取 +12% | 稀有 | 加速升级 |
| 护甲 | 每次受伤减少 1 点伤害（最低 1 点） | 稀有 | 对 BOSS 高伤有效 |
| 幸运 | 道具掉落概率 +15% | 稀有 | 影响掉落判定 |
| 再生 | 每 600 帧（~30 秒）恢复 1 HP | 史诗 | 缓慢但持久 |
| 连锁反应 | 击杀敌机 15% 概率对附近敌机造成 50% 伤害 | 史诗 | 清场利器 |

### 2.3 特殊（Special）—— 改变战斗方式的独特效果

| 特殊 | 效果 | 稀有度 | 说明 |
|------|------|--------|------|
| 弹射 | 子弹击中后 30% 概率弹射到附近敌机 | 稀有 | 单体→群体 |
| 穿透 | 基础子弹穿透 +1 个敌机 | 稀有 | 额外穿透（基础武器 Lv5 自带的不计入） |
| 击退 | 所有武器攻击附带小距离击退 | 稀有 | 生存向 |
| 灼烧 | 所有武器命中附加灼烧（3 秒内额外 20% 伤害） | 史诗 | DOT 叠加 |
| 冰冻 | 所有武器命中 10% 概率冻结敌机 30 帧 | 史诗 | 控场 |
| 吸血 | 击杀敌机 3% 概率恢复 1 HP | 史诗 | 续航 |
| 时间延长 | 所有临时 Buff 持续时间 +30% | 稀有 | 让道具 Buff 更持久 |

### 2.4 武器进化（Evolution）

当两个特定武器都升到 Lv5 时，可选进化为**超武**。进化后释放两个武器槽位，获得 1 个超强武器。

| 组合 A (Lv5) | 组合 B (Lv5) | 进化结果 | 效果 |
|-------------|-------------|---------|------|
| 追踪导弹 | 闪电链 | **雷霆风暴** | 导弹命中后触发 3 条连锁闪电 |
| 护盾环绕 | 火焰光环 | **烈焰星环** | 环绕球体变大 + 灼烧轨迹 + 击退 |
| 爆裂弹 | 冰冻弹 | **冰爆弹** | 爆炸冻结范围内所有敌机 60 帧 |
| 激光炮 | 僚机 | **双子激光** | 僚机也发射激光束 |
| 基础武器 | — | **歼灭光束** | 基础武器 Lv5 独有进化：5 路穿透 + 命中灼烧 |

## 三、升级选择 UI

### 3.1 界面布局

升级时游戏暂停，画布上绘制选择界面：

```
┌─────────────────────────────────────┐
│                                     │
│          ★ LEVEL UP! ★              │  ← 金色标题 + 发光
│           Lv.5 → Lv.6              │
│                                     │
│  ┌──────────┐┌──────────┐┌──────────┐  ← 3 张卡片
│  │  [图标]  ││  [图标]  ││  [图标]  │
│  │          ││          ││          │
│  │  追踪    ││  伤害    ││  护甲    │
│  │  导弹    ││  增幅    ││          │
│  │  Lv1     ││  +15%   ││  +1     │
│  │  NEW!    ││  普通    ││  稀有   │
│  └──────────┘└──────────┘└──────────┘
│                                     │
│       [刷新 ✕2]                    │  ← 刷新按钮（剩余次数）
│                                     │
└─────────────────────────────────────┘
```

### 3.2 卡片信息

- **图标**：Canvas 绘制简笔符号/形状（不需要美术素材）
- **名称**：i18n key
- **简短描述**：i18n key（如"发射追踪导弹"）
- **等级/数值**：如 "Lv1"、"+15%"
- **稀有度边框颜色**：普通=灰白、稀有=蓝色、史诗=紫色、传说=金色
- **"NEW!" 标记**：首次获得该武器/被动时显示

### 3.3 刷新机制

- 每次升级有刷新次数（普通=3/中等=2/困难=1），将 3 个选项全部替换为新随机项
- 刷新不消耗资源，但次数有限
- 来源：守卫陈塘关的"重燃币刷新"机制

### 3.4 交互

- **PC**：鼠标点击卡片
- **移动端**：手指点击卡片
- 点击卡片区域外不做任何操作（防止误触跳过选择）
- 选择后立即生效，游戏恢复

## 四、升级池与出牌逻辑

### 4.1 技能池定义（UpgradeDef）

```typescript
interface UpgradeDef {
  id: string;                    // 唯一标识
  type: "weapon" | "passive" | "special";
  rarity: "common" | "rare" | "epic" | "legendary";
  maxLevel: number;              // 最大等级（武器=5，被动/特殊=1或可叠加）
  weaponSlot: boolean;           // 是否占用武器槽位
  prerequisites: string[];       // 前置升级 id（如暴击伤害需先有暴击率）
  evolutionFrom: [string, string] | null;  // 进化来源（两个 Lv5 武器 id）
  label: TextKey;                // 名称 i18n key
  description: TextKey;          // 描述 i18n key
  icon: string;                  // 图标标识（Canvas 绘制用）
}
```

### 4.2 出牌算法

升级时从池中随机抽取 3 个选项：

**Step 1：收集当前可选项**
- 基础武器升级（未满级时）
- 已有武器升级（未满级时）
- 新武器（武器槽未满时，Lv3+ 开始出现）
- 被动（始终可出现）
- 特殊（Lv5+ 开始出现）
- 进化（前置武器均 Lv5 时出现，稀有度=传说）
- 排除：前置条件未满足的、已满级的、武器槽满时的新武器

**Step 2：按稀有度加权随机抽取 3 个**

基础权重（普通难度）：

| 稀有度 | 1-5 级 | 6-15 级 | 16+ 级 |
|--------|--------|---------|--------|
| 普通 | 60% | 40% | 25% |
| 稀有 | 30% | 35% | 40% |
| 史诗 | 8% | 18% | 25% |
| 传说 | 2% | 7% | 10% |

难度修正（叠加到基础权重上）：

| 难度 | 稀有权重加成 | 史诗权重加成 |
|------|------------|------------|
| 普通 | 0 | 0 |
| 中等 | -5% | -3% |
| 困难 | -10% | -8% |

**Step 3：保证规则**
- 3 个选项中保证至少 1 个武器类（如果有可用武器）
- 3 个选项不允许出现相同 id

### 4.3 已获得升级的存储

Hero 类新增属性：

```typescript
weapons: Map<string, number>;    // weaponId → level (1-5)
passives: Map<string, number>;   // passiveId → 叠加层数
specials: Set<string>;           // specialId（不可叠加）
```

## 五、敌机成长体系

### 5.1 设计原则

| 原则 | 说明 |
|------|------|
| TTK 锚定 | 各类敌机的击杀时间应有合理区间 |
| 功能定位不变 | 小型=炮灰、中型=中坚、大型=精英、精英=小BOSS |
| 敌机不削弱玩家 | 继承现有设计哲学 |
| 成长与玩家等级挂钩 | 后期不因 Build 成型而变无聊 |
| 新增精英敌机 | 扩充多样性，支撑 50 级游戏时长 |

### 5.2 TTK 目标（普通难度 + 中等 Build DPS≈25）

| 敌机类型 | TTK 目标 | 理由 |
|---------|---------|------|
| 小型 | 0.5-1 秒 | 保持一击即杀爽感，后期需 1-2 发 |
| 中型 | 2-3 秒 | 有存在感但不拖沓 |
| 精英 | 4-6 秒 | 小 BOSS 体验 |
| 大型 | 5-8 秒 | 持续集火 |
| BOSS | 30-60 秒 | 多阶段持久战 |

### 5.3 缩放参数调整

**核心变化**：幂函数指数从 1.1 → **1.3**，hpScale 翻倍。

| 敌机 | hpScale（旧→新） | speedScale | scoreScale | 设计意图 |
|------|-----------------|------------|------------|---------|
| 小型 | 0 → **0.008** | 0.012 | 0.03 | 1 级 1HP，10 级后约 2HP，有伤害被动仍可一击杀 |
| 中型 | 0.035 → **0.07** | 0.008 | 0.05 | 30 级 HP≈60（4×），50 级 HP≈125（8×） |
| 精英 | 新增 **0.08** | 0.010 | 0.07 | 8 级后出现，中型→大型之间过渡 |
| 大型 | 0.045 → **0.09** | 0.005 | 0.06 | 30 级 HP≈370（5.3×），50 级 HP≈860（12×） |

### 5.4 HP 精度调整

旧：`this.lives = Math.max(1, Math.ceil(this.lives))` — 取整导致 1.4→2 跳跃太大

新：`this.lives = Math.max(1, this.lives)` — 保留浮点数，伤害也支持小数

- 血量条显示取整，实际判定用浮点数
- HP=1.4 时：基础伤害 1 需 2 发，但伤害 1.5+ 只需 1 发，过渡更平滑

### 5.5 碰撞伤害分级

当前敌机碰撞伤害固定为 1 点，新系统下按类型分级：

| 敌机类型 | 碰撞伤害 | 说明 |
|---------|---------|------|
| 小型 | 1 | 不变 |
| 中型 | 1 | 不变 |
| 精英 | 2 | 高威胁，需优先躲避 |
| 大型 | 2 | 高威胁 |
| BOSS | 3 | 极度危险 |

配合护甲被动（"每次受伤减少 1 点，最低 1 点"）：
- 对小型/中型：护甲 = 减半伤害
- 对精英/大型：护甲减 1 点仍受 1 点
- 对 BOSS：护甲减 1 点仍受 2 点

### 5.6 新增精英敌机（Elite）

| 属性 | 值 | 说明 |
|------|---|------|
| 基础 HP | 40 | 中型（15）和大型（70）之间 |
| 基础速度 | 3 | 中型（4）和大型（2）之间 |
| 基础分数 | 50 | 中型（20）和大型（100）之间 |
| 基础经验 | 50 | 中型（20）和大型（100）之间 |
| 出现条件 | 玩家 **8 级** 后 | 中后期压力源 |
| 出现频率 | 基础 3%，每级 +0.3%，上限 10% | 低于中型，高于大型 |
| 道具掉落 | 与大型敌机同档位 | 护盾/火力/散弹/回血均可能 |
| 外观 | 复用 enemy3 图片 + `ctx.scale(0.7)` | 缩小 70% 的大型敌机 |

**俯冲行为（Dive）**——精英敌机的独特移动模式：

```
阶段1（距玩家远）：正常速度下落 + 小幅左右摆动
阶段2（进入玩家上方 200px 范围）：加速 2× 直线俯冲
阶段3（俯冲至玩家 Y 坐标后）：恢复正常速度，飞出屏幕
```

### 5.7 敌机生成节奏

#### 生成比例随等级变化

| 等级段 | 小型权重 | 中型权重 | 大型概率 | 精英概率 | 节奏感 |
|--------|---------|---------|---------|---------|--------|
| 1-7 | 高（15→10） | 低（5→7） | 极低（5%） | 0% | 炮灰海，轻松上手 |
| 8-15 | 中（10→6） | 中（7→10） | 低（5→7%） | 低（3→5%） | 引入精英，开始有压力 |
| 16-25 | 低（6→3） | 高（10→14） | 中（7→9%） | 中（5→7%） | 中型为主+精英点缀 |
| 26-35 | 极低（3→2） | 高（14→16） | 中高（9→11%） | 中高（7→8%） | 强敌密集 |
| 36-50 | 极低（2→1） | 极高（16→18） | 高（11→13%） | 高（8→10%） | 精英满天飞 |

#### 生成间隔随等级缩短

| 等级段 | 生成间隔（帧） | 每秒敌机数 | 理由 |
|--------|--------------|-----------|------|
| 1-10 | 8 | ~2.5 | 入门节奏 |
| 11-20 | 6-7 | ~3 | 火力提升后需更多目标 |
| 21-30 | 5-6 | ~3.5 | 武器多了需喂子弹 |
| 31-40 | 4-5 | ~4 | 满配需密度支撑 |
| 41-50 | 3-4 | ~4.5 | 终局密度 |

实现方式：`heroConfig.enemySpawnInterval` 改为等级相关动态值。

### 5.8 全难度敌机 HP 推演（中型敌机）

**计算公式**：`base × (1 + hpScale × scalingMul × (level-1)^1.3) × hpMul`

| 等级 | 普通 | 中等(×1.35/×1.15) | 困难(×1.7/×1.3) |
|------|------|-------------------|-----------------|
| 1 | 15 | 20 | 26 |
| 10 | 22 | 32 | 43 |
| 20 | 36 | 54 | 73 |
| 30 | 60 | 92 | 128 |
| 40 | 88 | 139 | 197 |
| 50 | 125 | 204 | 296 |

**TTK 推演**（中等 Build DPS≈25）：

| 等级 | 普通 | 中等 | 困难 |

### 6.2 新增原则

### 6.3 新 DifficultyConfig 接口

### 6.7 三档体验定位

**普通——"爽快体验"**：
- 敌机正常强度，3 次刷新几乎总能选到想要的
- 稀有/史诗出现概率正常，Build 成型快
- 体验：轻松构建理想 Build，享受满屏清敌爽感

- 敌机 HP +35%，碰撞伤害 +20%，2 次刷新需取舍

**困难——"生存考验"**：
- 敌机 HP +70%，碰撞伤害 +50%，仅 1 次刷新必须从 3 个中选

## 七、BOSS 系统

### 7.1 触发机制

| 条件 | 说明 |
|------|------|
| 首次触发 | 玩家达到 **5 级** |
| 后续触发 | 每隔 **5 级**（10/15/20/25/30/35/40/45/50 级） |
| 预警 | BOSS 出现前 **3 秒**（60 帧），屏幕顶部显示 "⚠ BOSS 来袭!" + 警报音效 |
| 特殊 | BOSS 出现时暂停敌机生成，屏幕上普通敌机加速飞出 |

### 7.2 BOSS 实体设计

BOSS 是独立的类（`Boss`），不继承 `Enemy`。

| 属性 | 说明 |
|------|------|
| HP | 根据等级和难度缩放 |
| 大小 | 画布宽度的 30-40%，固定在屏幕上方 1/4 区域水平巡逻 |
| 移动 | 水平缓慢来回 |
| 攻击阶段 | 根据 HP 比例切换（>60% / 30-60% / <30%） |
| 血量条 | 屏幕最顶部全宽，带 BOSS 名称 |
| 击败奖励 | 大量经验 + 保底稀有以上升级选项 |

BOSS HP 缩放公式：
```
实际HP = baseHP × (1 + 0.3 × (level/5 - 1)) × 难度HP乘数 × 难度BOSS HP乘数
```

## 九、伤害公式调整

当前：`damage = (baseDamage + extraDamage) × firepowerMultiplier`

新公式：
```
damage = (baseDamage + extraDamage) × (1 + passiveDamageBonus) × firepowerMultiplier × critMultiplier
```

- `baseDamage`：子弹基础伤害（bulletConfig.baseDamage = 1）
- `extraDamage`：基础武器升级带来的伤害加成
- `passiveDamageBonus`：所有"伤害增幅"被动的累计值（如 3 个 +15% = +45%）
- `firepowerMultiplier`：火力道具 Buff 倍率（激活时 2，否则 1）
- `critMultiplier`：暴击时 = 暴击倍率（基础 2× + 暴击伤害被动），否则 = 1

额外武器的伤害独立计算，但共享被动加成和暴击。

## 十三、关键风险与缓解

| 风险 | 缓解措施 |
|------|---------|
| 武器特效绘制性能（多武器同时发射） | 所有弹幕使用对象池，统一批量绘制；限制同屏弹幕数上限 |
| 升级选择打断游戏节奏 | 暂停时间短（选完即恢复）；连续升级时逐个弹出 |
| BOSS 弹幕密度过高导致卡顿 | BOSS 弹幕用简单圆形，不用图片；限制同屏敌机弹幕<50 |
| 数值平衡（某些 Build 过强/过弱） | 所有数值集中在 config.ts，可快速调整；先实现后测试迭代 |
| 武器视觉效果缺少美术素材 | Canvas 绘制简笔图形（圆形导弹/线段激光/弧形光环），零资源依赖 |
| 敌机 HP 浮点数显示问题 | 血量条/伤害数字取整显示，内部判定用浮点 |
| 难度乘数与新成长曲线叠加爆炸 | ✅ 已缓解：hpMultiplier 从 1.0/1.6/2.2 降至 1.0/1.3/1.6；scalingMultiplier 从 1.0/1.4/1.8 改为 1.0/1.5/2.0（成长加速而非简单乘 HP） |
| 升级选择 UI 在小屏手机上卡片过小 | 卡片尺寸按 fontScale 缩放，最小宽度不低于 80px |

## 十四、与现有约束的兼容性

本设计遵守 PROJECT_GUIDE.md 和 project_memory.md 中的所有硬约束，特别关注：

| 约束 | 本设计如何遵守 |
|------|--------------|
| 逆序遍历修改数组 | 新增的 EnemyBullet、升级选项等数组遍历均使用逆序 |
| `>=` 而非 `===` 做循环重置 | BOSS 阶段切换等计数器使用 `>=` |
| onLoadComplete 一次性调用 | 不涉及资源加载变更 |
| fontScale 缩放 | 升级 UI、BOSS 血条等新增 UI 均乘以 fontScale |
| Canvas 用逻辑尺寸 | 新增渲染均使用 `width`/`height` 而非 `canvas.width/height` |
| i18n t() 函数 | 所有新增文本使用 TextKey + t() |
| 配置文件存储 TextKey | 升级名称/描述/BOSS 名称均为 TextKey |
| 纯数字不翻译 | 伤害数值、HP 数值等直接拼接 |
| import 路径用 .js 后缀 | 新增模块间 import 使用 .js 后缀 |
| `export let` 不重复导出 | 新增模块注意不重复导出 |
| 难度只增强敌机不削弱玩家 | 新增维度（刷新次数/稀有度权重）是"选择空间"而非"属性削弱" |
| HP 下限 `Math.max(1, ...)` | 敌机 HP 保留浮点但仍 `Math.max(1, ...)` 保底 |
